FAQ

Absolutely! The data between client <-> central server and node <-> central server is end-to-end encrypted. It's no matter you're running cloud or hosted version plaintext session data never hits central server. So if you prefer self-hosted it's safe to install it to public cloud.
From the client side extra security is provided by 2-factor authentication.

We use gRPC as control protocol and NATS for session streaming. The local communication is built on HTTP/Websockets.

Keys are generated indepedently on client and nodes. Loosing client's key will have no impact. But node's key must be stored safely as it is used to encrypt and decrypt session data. Loosing node's key means you will lost your session history. You may generate node key beforehand or leave it for node itself.

You may store service configuration either in file or in database. If you choose database you're able to edit config right from GUI. For better security config itself is stored encrypted in DB. Only node can read it using its key.

If you edit config using GUI graceful reload is performed. Otherwise you can send USR1 signal to process and it will lead to graceful reload.

Traditionally connection to host means connection via SSH and running default interpreter e.g. Bash. In Teamshell this default behaviour is preserved but not limited to it. You may define any other command as entry point, for instance: psql, htop, tail, etc. Once you exit this command the session ends. That can act as an additional limitation measure while providing access to sensitive hosts.

For GUI Linux, MacOS and Windows are supported. TUI can run on Linux, MacOS and WSL on Windows.
Node can run Linux (including Docker/k8s)
